# Módulo 5 - SOLID y Clean Code

## Objetivo

Aprender a diseñar software mantenible, escalable y fácil de modificar aplicando principios SOLID y Clean Code.

## Temas

- Clean Code
- Principio de Responsabilidad Única (SRP)
- Principio Abierto/Cerrado (OCP)
- Sustitución de Liskov (LSP)
- Segregación de Interfaces (ISP)
- Inversión de Dependencias (DIP)
- Inyección de Dependencias
- Refactorización
- Code Smells
- Arquitectura por capas

## Proyecto

Aplicar SOLID en:

- ClienteService
- CuentaService
- TransaccionService

## Reglas

- Una clase debe tener una única responsabilidad.
- Evitar duplicación de código.
- Favorecer composición sobre herencia.
- Programar contra interfaces.
- Mantener métodos cortos y claros.
- Utilizar nombres descriptivos.

## Al finalizar

El estudiante será capaz de identificar y corregir malas prácticas aplicando SOLID y Clean Code en proyectos Spring Boot.
