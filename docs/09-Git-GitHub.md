# Módulo 9 - Git, GitHub y GitFlow

## Objetivo

Aprender a trabajar con Git y GitHub utilizando un flujo de trabajo profesional similar al usado en equipos de desarrollo.

## Temas

- Control de versiones
- Repositorios locales y remotos
- GitHub
- GitFlow
- Conventional Commits
- Branching
- Merge
- Rebase
- Pull Request
- Code Review
- Resolución de conflictos
- GitHub Actions (Introducción)

## Proyecto

Gestionar el desarrollo del sistema bancario utilizando GitFlow.

## Flujo de ramas

- main
- develop
- feature/*
- release/*
- hotfix/*
- bugfix/*

## Conventional Commits

- feat:
- fix:
- docs:
- test:
- refactor:
- chore:

Ejemplos:

feat(clientes): create client endpoint

fix(cuentas): validate negative balance

docs(readme): update installation guide

## Reglas

- No trabajar directamente en main.
- Una funcionalidad por rama.
- Un commit por cambio lógico.
- Crear Pull Request antes de integrar.
- Escribir mensajes de commit claros.
- Resolver conflictos antes de fusionar.

## Al finalizar

El estudiante será capaz de trabajar con Git y GitHub siguiendo un flujo profesional utilizado en empresas.
