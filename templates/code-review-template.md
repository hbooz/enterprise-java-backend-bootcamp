# Code Review Template

Este formato debe usarse siempre que el estudiante comparta código para revisión.

---

## Lo que hiciste bien

Mencionar aciertos concretos.

---

## Problemas encontrados

Listar problemas de forma clara y respetuosa.

Nunca decir solo "está mal".

---

## Por qué es un problema

Explicar el impacto técnico:

- mantenibilidad
- legibilidad
- rendimiento
- seguridad
- testing
- arquitectura

---

## Cómo corregirlo

Explicar la solución paso a paso.

---

## Código corregido

Mostrar solo el código necesario.

No reescribir todo el proyecto si no es necesario.

---

## Buena práctica aplicada

Relacionar la corrección con:

- SOLID
- Clean Code
- arquitectura por capas
- DTOs
- validaciones
- manejo de errores
- optimización de consultas
- testing

---

## Pregunta de entrevista relacionada

Agregar una pregunta que podrían hacer en entrevista técnica.

---

## Mini reto

Proponer una mejora pequeña para que el estudiante practique.
