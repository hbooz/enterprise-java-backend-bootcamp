# Session Template

Todas las sesiones del bootcamp deben seguir exactamente este formato.

No omitir secciones.

---

# Objetivo del día

Explicar claramente qué aprenderá el estudiante.

---

# Prerrequisitos

Indicar qué conocimientos debe dominar antes de comenzar.

Si no cumple los prerrequisitos, indicarlo y recomendar repasar el tema correspondiente.

---

# Qué aprenderás hoy

Lista corta de conceptos.

Máximo tres conceptos importantes.

---

# Explicación simple

Responder siempre:

- ¿Qué es?
- ¿Para qué sirve?
- ¿Cuándo se usa?
- ¿Por qué existe?

Evitar lenguaje excesivamente técnico.

---

# Analogía

Usar una analogía sencilla cuando ayude a comprender el concepto.

---

# Ejemplo empresarial

Todos los ejemplos pertenecen al sistema bancario.

---

# Arquitectura

Antes del código explicar:

- dónde vive esta clase
- qué responsabilidad tiene
- con qué otras clases interactúa

---

# Código guiado

Construir el código paso a paso.

Nunca mostrar una solución enorme sin explicación.

Después de cada bloque explicar qué hace.

---

# Explicación del código

Explicar:

- responsabilidades
- flujo de ejecución
- buenas prácticas
- errores comunes

---

# Ejercicio práctico

El estudiante debe escribir código.

No resolver inmediatamente.

Esperar el intento del estudiante.

---

# Mini reto

Plantear un reto ligeramente más complejo.

---

# Preguntas de reflexión

Hacer entre 2 y 5 preguntas.

El estudiante debe responder antes de continuar.

---

# Errores comunes

Explicar:

- qué error ocurre
- por qué ocurre
- cómo solucionarlo

---

# Buenas prácticas

Relacionar el tema con:

- SOLID
- Clean Code
- Arquitectura por capas
- Mantenibilidad
- Escalabilidad
- Rendimiento

---

# Testing

Si aplica al tema:

- ¿Qué probar?
- ¿Cómo probar?
- ¿Qué tipo de prueba usar?
- ¿Qué casos cubrir?

---

# OpenAPI

Si se crea un endpoint:

Recordar documentarlo usando OpenAPI.

---

# Git

Indicar:

Rama GitFlow

Commit recomendado

Archivos modificados

---

# Checklist

□ Entendí el concepto

□ Escribí el código

□ Ejecuté el proyecto

□ Probé la funcionalidad

□ Documenté el endpoint

□ Subí cambios a GitHub

---

# Preguntas tipo entrevista

Generar entre 3 y 5 preguntas relacionadas con el tema.

---

# Próxima sesión

Indicar cuál será el siguiente tema.

No avanzar automáticamente.

Esperar confirmación del estudiante.
